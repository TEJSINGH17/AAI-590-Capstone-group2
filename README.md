# AAI-590 Capstone Group 2

Traffic sign detection and HUD publishing workflow built around YOLOv8 and the
LISA traffic sign dataset.

## Project Structure

`application/`
Runtime scripts for publishing detections to the HUD pipeline.

`models/`
Base model weights used as training starting points, such as `yolov8n.pt`.

`trained_models/`
Output directory for fine-tuned model checkpoints copied from training runs.

`training/`
Dataset preparation, class definitions, training scripts, and the training
notebook.

`training_data/`
Raw source datasets. The expected LISA dataset root is
`training_data/Lisa/`.

`test_data/`
Sample input and output videos for inference testing.

`runs/`
Generated artifacts from Ultralytics training and local inference runs.

## Training Workflow

1. Place the raw LISA dataset under `training_data/Lisa/`.
2. Convert annotations into YOLO format:

```bash
python -m training.prepare_lisa \
  --lisa_root training_data/Lisa \
  --out_root training/lisa_yolo
```

3. Train YOLOv8 on the converted dataset:

```bash
python -m training.train_yolov8 \
  --data training/lisa_yolo/dataset.yaml \
  --base_model models/yolov8n.pt
```

4. Run inference on a test video:

```bash
python test_mp4_yolov8.py \
  --model trained_models/yolov8n_lisa_best.pt \
  --input test_data/output_ds_1.mp4 \
  --output runs/annotated/out.mp4
```

## Notes

- `training/lisa_yolo/` is generated by `prepare_lisa.py` and is ignored by git.
- `runs/` contains generated outputs and should not be treated as source code.
- `trained_models/.gitkeep` is committed so the output directory exists before
  training.
